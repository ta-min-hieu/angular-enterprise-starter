# Internationalization (i18n)

## Purpose

Quản lý văn bản hiển thị tập trung, không hardcode text trong Component/Template.

Hỗ trợ đa ngôn ngữ, chuyển ngữ ngay tại runtime (không cần rebuild/reload riêng theo locale).

---

# Library (đã chốt)

`@jsverse/transloco`.

Lý do chọn thay vì `@angular/localize`:

`@angular/localize` biên dịch tĩnh theo locale (mỗi locale một bundle riêng, đổi ngôn ngữ phải điều hướng sang bundle khác) — không phù hợp với yêu cầu chuyển ngữ tức thời qua một nút bấm, giống cách `ThemeService` đổi Light/Dark hiện có.

`@jsverse/transloco` là giải pháp runtime, tích cực bảo trì, tương thích tốt với Standalone Component.

Không dùng song song `@angular/localize` và `@jsverse/transloco`.

---

# Ngôn ngữ hỗ trợ

`vi` (mặc định)

`en`

Thêm ngôn ngữ mới: thêm entry vào `availableLangs` trong `core/i18n/transloco.config.ts` và tạo file dịch tương ứng.

---

# File dịch

Đặt tại `public/i18n/{lang}.json` (không đặt trong `src/`), tải qua HTTP tại runtime, không bundle vào JS.

Cấu trúc: object phẳng, key phân cấp bằng dấu chấm, namespace theo Feature/Shared/Core:

```json
{
  "common.actions.save": "Lưu",
  "common.actions.cancel": "Huỷ",
  "products.title": "Sản phẩm",
  "products.form.name.label": "Tên sản phẩm"
}
```

`vi.json` và `en.json` phải luôn có cùng tập key. Thiếu key ở một ngôn ngữ là lỗi cần sửa ngay, không để sót.

---

# Sử dụng trong Component

Template: dùng `TranslocoPipe` (`| transloco`), không nối chuỗi thủ công.

Component: nếu cần lấy text trong TypeScript (ví dụ message cho `nz-modal`/`nz-notification`), inject `I18nService` (xem mục Adapter), không inject thẳng `TranslocoService` ở nơi dùng cuối.

---

# Adapter (đã chốt)

Theo `18-dependency-management.md` — Third-party Library có khả năng phải thay thế trong tương lai nên được bọc qua Adapter.

`core/i18n/i18n.service.ts` bọc `TranslocoService`, expose:

`locale` — Signal ngôn ngữ hiện tại.

`availableLocales` — danh sách ngôn ngữ hỗ trợ (label + code) để dựng UI chuyển ngữ.

`setLocale(code)` — đổi ngôn ngữ, tự lưu lựa chọn (giống cơ chế lưu Theme).

Code trong Feature chỉ phụ thuộc `I18nService`, không import trực tiếp `TranslocoService`/`TranslocoPipe` ngoài phạm vi `core/i18n` và các Component dùng pipe trong template.

---

# Không hardcode text

Mọi text hiển thị cho người dùng (label, placeholder, message lỗi, tiêu đề trang, nội dung Modal/Notification/Toast...) phải đi qua key dịch.

Ngoại lệ: tên riêng (thương hiệu), giá trị dữ liệu (tên sản phẩm do người dùng nhập).

---

# Checklist

✓ Không text tiếng Việt/Anh hardcode trong Template/Component

✓ `vi.json` và `en.json` đồng bộ key

✓ Text mới thêm phải có key ở cả hai ngôn ngữ trước khi merge

✓ Dùng qua `I18nService`/`TranslocoPipe`, không gọi thẳng `TranslocoService` ngoài `core/i18n`
